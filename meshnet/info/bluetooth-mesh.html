Bluetooth Mesh erklärt

Bluetooth Mesh ermöglicht vernetzte Kommunikation zwischen vielen Bluetooth-Geräten — ohne Server.

Bluetooth Classic vs BLE Mesh

MeshNet nutzt BLE (Bluetooth Low Energy) Mesh — nicht Classic Bluetooth:

  • BLE = niedrigere Energie, mehr Geräte, mesh-fähig
  • Classic = point-to-point (1:1 Verbindung)
  • BLE Mesh = many-to-many (viele:viele)

Web Bluetooth API

Die Web Bluetooth API erlaubt Browser-Access zu BLE-Geräten:

  • navigator.bluetooth.requestDevice() — Geräte scannen
  • Chrome/Edge: native Support (Android + Desktop)
  • Safari/Firefox: kein Support
  • Festival: Android ~70%, iOS ~25%, Desktop ~5%

Mesh Topologie

Star Topologie

Alle verbinden mit einem Hub. Hub fällt aus = Netzwerk fällt aus.

Mesh Topologie

Alle verbinden mit Nachbarn. Node fällt aus = Netzwerk reroutet.

Managed Flooding

Das Routing-Verfahren in MeshNet:

  • Nachricht wird an alle Nachbarn gesendet
  • Relay Nodes forwarding weiter
  • TTL (Time To Live) begrenzt die Reichweite
  • Route tracking prevents loops

Reichweite

Bluetooth Reichweite ~10m. Multi-hop erweitert das Netzwerk:

  • 1 Hop = direkte Verbindung (~10m)
  • 7 Hops = ~70m in linearer Anordnung
  • Mesh-Topologie = viel mehr coverage
  • Festival-Gelände: 10-30 Nodes möglich
