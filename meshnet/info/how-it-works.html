Wie MeshNet funktioniert

Die Canvas-Visualisierung auf der Startseite zeigt das unsichtbare Bluetooth-Netzwerk in Echtzeit.

Nodes

Jedes Gerät mit MeshNet ist ein Node. Nodes sind entweder Relay oder Leaf:

  • Relay Nodes (grün) — forwarding messages, größere Darstellung
  • Leaf Nodes (blau) — receive/send only, kleinere Darstellung
  • Dieses Gerät (grün mit glow) — immer Relay, dein Node

Verbindungen

Linien zwischen Nodes zeigen Bluetooth-Reichweite (~10m). Nodes innerhalb der Reichweite können direkt kommunizieren.

Message Routing — Managed Flooding

Wenn du eine Nachricht sendest:

  • Yellow packets zeigen die Route durch das Netzwerk
  • Relay Nodes forwarding die Nachricht (TTL-basiert)
  • Max 7 Hops — Nachrichten sterben nach 7 Relay-Schritten
  • BFS berechnet die Hop-Distanz von deinem Node zu allen anderen

Simulation vs Bluetooth

Simulation

Simulated Nodes bewegen sich auf der Canvas. Messages werden geroutet. Funktioniert auf allen Geräten.

Bluetooth

Reale Nodes via Web Bluetooth API. Android Chrome/Edge. Echte Mesh-Kommunikation.

Self-Healing

Wenn Nodes offline gehen, reroutet das Netzwerk automatisch. BFS berechnet neue Hops. Das Netzwerk heilt sich selbst.
